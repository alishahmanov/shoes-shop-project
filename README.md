# Shoes Shop Project

Полнофункциональный интернет-магазин обуви с современным дизайном, адаптивной версткой, интеллектуальной системой фильтрации и корзиной покупок.

## Описание проекта

Shoes Shop — это веб-приложение интернет-магазина обуви, разработанное с использованием современных веб-технологий. Проект представляет собой полноценный сайт электронной коммерции с 10 функциональными страницами, системой управления корзиной, продвинутой фильтрацией товаров, формой индивидуального заказа и интерактивными элементами.

### Основные возможности

**Каталог и товары**

- 16 моделей обуви различных брендов (Nike, Adidas, Puma, Jordan, New Balance, Reebok, Asics, Vans, DC shoes, Crocs, Skechers, Under Armour)
- Детальные карточки товаров с изображениями, ценами и характеристиками
- Двухстраничная пагинация каталога с умным переключением

**Интеллектуальная фильтрация**

- Кросс-страничный поиск: автоматическое объединение товаров со всех страниц при активных фильтрах
- Множественная фильтрация по бренду (12 брендов), категории (4 типа), цвету (6 цветов) и цене
- Динамическое обновление без перезагрузки страницы
- Автоматическое скрытие пагинации при активных фильтрах

**Корзина покупок**

- Отдельная страница корзины с полным функционалом
- Добавление товаров из любой страницы каталога
- Управление количеством (увеличение/уменьшение)
- Автоматический расчет итоговой стоимости
- Удаление товаров из корзины
- Сохранение состояния в localStorage (данные сохраняются между сеансами)
- Счетчик товаров в header на всех страницах
- Toast-уведомления при добавлении товаров

**Поиск товаров**

- Автодополнение при вводе запроса
- ARIA-атрибуты для доступности
- Мгновенная фильтрация результатов
- Очистка поиска одним кликом

**Форма индивидуального заказа**

## Технологии

### Frontend

- **HTML5**: Семантическая разметка, ARIA-атрибуты для доступности, SEO-оптимизация
- **CSS3**: Flexbox, Grid Layout, CSS Variables, медиа-запросы, transitions, animations
- **JavaScript (ES6+)**: Модульная архитектура, async/await, Fetch API, localStorage, DOM manipulation

### Ключевые технологии и паттерны

- **Vanilla JavaScript**: Без использования фреймворков (React, Vue и т.д.)
- **Mobile-first**: Адаптивная верстка с приоритетом мобильных устройств
- **Progressive Enhancement**: Постепенное улучшение функциональности
- **Accessibility (a11y)**: ARIA-атрибуты, семантический HTML, keyboard navigation
- **LocalStorage API**: Сохранение корзины и настроек пользователя
- **Fetch API**: Асинхронная загрузка данных между страницами
- **IntersectionObserver**: Lazy loading изображений для оптимизации
- **CSS Grid & Flexbox**: Современная адаптивная верстка
- **Event Delegation**: Оптимизация обработки событий
- **Module Pattern**: Инкапсуляция функциональности

### Оптимизация

- Lazy loading изображений (IntersectionObserver)
- Оптимизация изображений (AVIF, JPEG)
- Минимизация reflow/repaint
- Кеширование данных в localStorage
- Асинхронная загрузка скриптов
- Responsive images с srcset
- CSS-переменные для переиспользования значений

**Кросс-страничная фильтрация:**

```javascript
// При загрузке страницы товары с обеих страниц объединяются
const loadOtherPageProducts = async () => {
  const response = await fetch("category-page2.html");
  const html = await response.text();
  // Парсинг и добавление товаров в общий массив
};
```

**Возможности:**

- Автоматическая загрузка товаров со второй страницы через Fetch API
- Фильтрация по нескольким параметрам одновременно (AND-логика)
- Поддержка множественных значений в одном фильтре (OR-логика)
- Динамическое скрытие пагинации при активных фильтрах
- Восстановление оригинальной пагинации при сбросе фильтров

**Доступные фильтры:**

- **Бренд** (12 вариантов): Nike, Jordan, Adidas, Puma, New Balance, Reebok, Asics, Vans, DC shoes, Crocs, Skechers, Under Armour
- **Категория** (4 варианта): Кроссовки, Кеды, Ботинки, Сабо
- **Цвет** (6 вариантов): Черный, Белый, Серый, Коричневый, Красный, Синий
- **Цена** (4 диапазона): Все, до 30 000 ₸, 30 000-45 000 ₸, свыше 45 000 ₸

**Параметры расчета:**

- **Базовая цена**: Зависит от выбранной модели (12 000 - 55 000 ₸)
- **Материал**: Кожа (+5 000 ₸), Замша (+7 000 ₸), Текстиль (0 ₸)
- **Стелька**: Стандарт (0 ₸), Ортопедическая (+2 000 ₸), Спортивная (+1 500 ₸)
- **Дополнительно**: Светоотражатели (+1 000 ₸), Водозащита (+3 000 ₸), Вентиляция (+2 500 ₸)
- **Доставка**: Самовывоз (0 ₸), Курьер (+1 500 ₸), Почта (+800 ₸), Экспресс (+3 000 ₸)
- **Количество**: Умножает стоимость товара

**Формула расчета:**

```
Итого = (Базовая цена + Материал + Стелька + Доп. опции) × Количество + Доставка
```

**Визуальные превью:**

- Предпросмотр выбранного цвета на модели обуви
- Загрузка и отображение референсного изображения

## Установка и запуск

### Требования

- Современный веб-браузер (Chrome, Firefox, Safari, Edge)
- Веб-сервер (для корректной работы fetch API)

### Локальный запуск

1. Клонируйте репозиторий:

```bash
git clone https://github.com/alishahmanov/shoes-shop-project.git
cd shoes-shop-project
```

2. Запустите локальный сервер (выберите один из вариантов):

**Python 3:**

```bash
python -m http.server 8000
```

**Node.js (http-server):**

```bash
npx http-server -p 8000
```

**PHP:**

```bash
php -S localhost:8000
```

**VS Code Live Server:**

- Установите расширение "Live Server"
- Откройте проект в VS Code
- Нажмите "Go Live" в статус-баре

3. Откройте в браузере:

```
http://localhost:8000
```

### Архитектура проекта

**Принципы:**

- Модульность: каждая страница имеет свой JS-модуль
- Разделение ответственности: HTML (структура), CSS (стили), JS (логика)
- Progressive Enhancement: работает без JavaScript (частично)
- Accessibility First: ARIA-атрибуты, семантический HTML
- Performance: lazy loading, минимизация DOM-операций

## Использование

### Добавление товара в корзину

1. Перейдите на страницу каталога или категории
2. Найдите нужный товар
3. Нажмите кнопку "В корзину"
4. Появится toast-уведомление об успешном добавлении
5. Счетчик в header обновится

### Фильтрация товаров

1. Откройте страницу категорий
2. В левой панели выберите нужные фильтры (бренд, категория, цвет, цена)
3. Товары отфильтруются автоматически со всех страниц
4. Пагинация скроется при активных фильтрах
5. Снимите фильтры для возврата к стандартному виду

### Работа с корзиной

1. Нажмите "Корзина" в header
2. Используйте кнопки +/− для изменения количества
3. Нажмите "×" для удаления товара
4. Итоговая сумма пересчитывается автоматически
5. Данные сохраняются при перезагрузке страницы

### Создание индивидуального заказа

1. Перейдите на страницу "Форма под заказ"
2. Выберите модель, размер, количество
3. Настройте цвет, материал, стельку
4. Отметьте дополнительные опции
5. Выберите способ доставки
6. Загрузите референсное изображение (опционально)
7. Заполните контактные данные
8. Итоговая цена рассчитывается автоматически

### Переключение темы

1. Найдите кнопку "Ночь/День" в header
2. Нажмите для переключения
3. Выбор сохранится автоматически

## Известные ограничения

1. **Fetch API требует веб-сервер**: Открытие через file:// не работает
2. **localStorage ограничен**: Максимум ~5-10 МБ
3. **Нет backend**: Данные форм не отправляются на сервер
4. **IE 11**: Не поддерживается (ES6+)
